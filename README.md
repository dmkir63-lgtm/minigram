# MiniGram

Учебный мини-мессенджер на Flask + SQLite + Flask-SocketIO.

## Что есть

- Регистрация и вход по `@тегу` или email.
- У пользователя отдельно:
  - ник / отображаемое имя;
  - уникальный тег `@username`.
- Личные сообщения без обязательного добавления в друзья.
- Статусы личных сообщений:
  - `✓` отправлено;
  - `✓✓` доставлено;
  - подсвеченные `✓✓` прочитано.
- В списке чатов показывается последнее сообщение и время.
- Секции `Личные чаты` и `Мои каналы` можно свернуть/развернуть.
- Настройка профиля: смена ника и выбор, кто может писать в личку:
  - все пользователи;
  - только друзья.
- Отдельная кнопка добавления пользователя в друзья.
- Кнопка удаления пользователя из друзей: чат остаётся, но доступ к отправке пересчитывается по настройкам приватности.
- Блокировка/разблокировка пользователя: чат остаётся в списке, но писать нельзя, если хотя бы одна сторона заблокировала другую.
- Список заблокированных пользователей в меню профиля с возможностью разблокировать.
- Удаление личного чата: чат пропадает из списка, а вся история между пользователями стирается из таблицы `messages`; пользователя потом можно снова найти через поиск и написать.
- Каналы:
  - название канала;
  - уникальный тег `@channel_tag`;
  - открытый или закрытый режим;
  - вступление в закрытый канал через заявку;
  - owner/admin могут принимать заявки;
  - owner может назначать и снимать админов;
  - owner может менять название и режим канала в настройках;
  - owner может удалить канал полностью из базы вместе с постами, участниками и заявками;
  - подписчик/админ может убрать канал из своих каналов с подтверждением.

## Запуск

```bash
pip install -r requirements.txt
python app.py
```

Открой:

```text
http://127.0.0.1:5000
```

Если SMTP не задан, код подтверждения регистрации печатается в консоль Flask.

## Переменные окружения для почты

```bash
export SMTP_USER="your@gmail.com"
export SMTP_PASS="google_app_password"
```

## Миграции

Приложение само добавляет недостающие поля в старую SQLite-базу:

- `users.display_name`
- `users.pm_privacy`
- `channels.username`
- `channels.is_private`
- `channel_members.role`
- таблицу `channel_join_requests`
- таблицу `user_blocks`
- таблицу `hidden_private_chats`
- `messages.delivery_status`
- `messages.delivered_at`
- `messages.read_at`

Старые каналы остаются открытыми, старые пользователи получают режим лички `everyone`.

## Серверное шифрование базы данных

В проект добавлен режим шифрования базы через SQLCipher.

По умолчанию проект запускается как раньше и использует обычную SQLite-базу:

```bash
python app.py
```

Чтобы включить шифрование, установите дополнительную зависимость:

```bash
pip install -r requirements-encrypted.txt
```

Задайте ключ базы данных:

```bash
export DB_ENCRYPTION_KEY="очень-длинный-секретный-ключ"
```

После этого приложение будет использовать файл:

```text
minigram_encrypted.db
```

Если ключ задан, но открыть базу нельзя, значит ключ неверный или файл был создан другим ключом.

### Как работает шифрование

- файл базы на диске хранится в зашифрованном виде;
- при запуске Flask передаёт ключ в SQLCipher через `PRAGMA key`;
- при `SELECT` SQLCipher сам расшифровывает нужные страницы базы в памяти;
- Flask получает обычные строки и отправляет их пользователю через HTTP/WebSocket;
- при `INSERT`/`UPDATE` SQLCipher сам шифрует данные перед записью на диск.

Это серверное шифрование, не end-to-end encryption: сервер всё ещё видит сообщения в расшифрованном виде во время обработки.

### Как зашифровать уже существующую базу

Если у вас уже есть обычная база `minigram.db`, сконвертируйте её в зашифрованную:

```bash
export DB_ENCRYPTION_KEY="очень-длинный-секретный-ключ"
python encrypt_existing_db.py minigram.db minigram_encrypted.db
```

Если `minigram_encrypted.db` уже существует и его нужно перезаписать:

```bash
python encrypt_existing_db.py minigram.db minigram_encrypted.db --force
```

После успешной проверки старую незашифрованную `minigram.db` лучше удалить или убрать из рабочей папки, иначе она останется plaintext-копией данных.

### Windows PowerShell

```powershell
$env:DB_ENCRYPTION_KEY="очень-длинный-секретный-ключ"
python app.py
```
